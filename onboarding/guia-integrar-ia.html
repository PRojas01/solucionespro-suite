¿Qué es esto, en simple?

Un asistente de IA (como Claude, Codex u opencode) es un programa que conversa contigo y puede hacer trabajo real en tus proyectos: escribir y revisar código, ordenar el proceso, documentar. Las herramientas de la Suite se conectan a ese asistente para darle control y método: que apruebes lo que cambia (DevControl), que siga un orden (devSteps), y más.

La idea clave: tú no tienes que programar todo a mano. Tú diriges, la IA propone y ejecuta, y tú apruebas los cambios importantes.
1

Prepara tu equipo una sola vez

Casi todos los asistentes de terminal se pueden instalar con Node.js, un programa base gratuito. Instala la versión LTS. Ollama es la excepción: no necesita Node.js.

Descarga oficial de Node.js NodeSource para Linux
Instalar Node.js - Windows
  1. Entra a nodejs.org/en/download y descarga la versión LTS para Windows.
  2. Abre el instalador y acepta la opción que agrega Node al PATH.
  3. Cierra y vuelve a abrir PowerShell.
  4. Verifica:
    node --version
    npm --version
Instalar Node.js - Mac
  1. Descarga la versión LTS en nodejs.org/en/download e instala el archivo .pkg.
  2. Si ya usas Homebrew, también puedes instalarlo así:
    brew install node
  3. Verifica en Terminal:
    node --version && npm --version
Instalar Node.js - Linux Ubuntu/Debian
  1. Opción simple desde NodeSource:
    curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - && sudo apt install -y nodejs
  2. Verifica:
    node --version && npm --version
  3. En Fedora, Arch u otra distro, usa el gestor de paquetes de tu sistema o las instrucciones oficiales de Node.js.
Nota de permisos: los comandos npm install -g instalan programas globales. Si aparece EACCES o “permission denied”, no adivines: en Mac/Linux usa un gestor como nvm o corrige permisos de npm; en Windows abre PowerShell como administrador solo si entiendes por qué.
2

Instala tu asistente de IA

Elige uno para empezar. Puedes tener varios después. Para empezar rápido: Claude Code o Codex. Si quieres IA local y privacidad: Ollama. Si quieres probar varios proveedores: opencode.

Claude Code de Anthropic

Asistente en tu terminal. Requiere Node.js 18 o superior y una cuenta compatible de Claude o Anthropic.

Guía oficial de Claude Code Comandos oficiales
  1. Instálalo:
    npm install -g @anthropic-ai/claude-code
  2. Verifica la instalación:
    claude --version
    claude doctor
  3. Entra a la carpeta de tu proyecto y arráncalo:
    cd ruta/de/tu/proyecto && claude
  4. Inicia sesión cuando lo pida. Puede usar Claude Pro/Max, Anthropic Console o una configuración empresarial.
Si falla: Anthropic recomienda no usar sudo npm install -g para Claude Code; si ves permisos, revisa la configuración de npm o usa el instalador recomendado por la guía oficial.

Codex de OpenAI

Asistente de OpenAI para trabajar desde terminal, editor o app. Puedes entrar con ChatGPT o usar API key.

Inicio oficial de Codex Repositorio oficial Paquete npm
  1. Instalador directo para Mac/Linux:
    curl -fsSL https://chatgpt.com/codex/install.sh | sh
  2. Instalador directo para Windows PowerShell:
    powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"
  3. Alternativa con npm si ya instalaste Node.js:
    npm install -g @openai/codex
  4. Verifica y entra:
    codex --version
    codex --login
  5. Arráncalo dentro de la carpeta de tu proyecto:
    cd ruta/de/tu/proyecto && codex
Si falla: si codex no aparece, cierra y abre la terminal; si sigue igual, revisa que la carpeta global de npm esté en el PATH o usa el instalador directo.

opencode multi-proveedor

Asistente de terminal que funciona con muchos proveedores, con ChatGPT/Copilot y también con modelos locales.

Documentación oficial de opencode Descargas oficiales
  1. Instalador recomendado en Mac/Linux/WSL:
    curl -fsSL https://opencode.ai/install | bash
  2. Alternativa con Node.js:
    npm i -g opencode-ai
  3. En Mac con Homebrew:
    brew install anomalyco/tap/opencode
  4. Verifica e inicia:
    opencode --version
    cd ruta/de/tu/proyecto && opencode
  5. Conecta proveedor con el comando /connect dentro de opencode, o usa Ollama local si quieres empezar sin pagar.
Windows: opencode recomienda WSL para una experiencia más estable. Si trabajas en Windows puro, usa la app de escritorio o prueba en WSL.

Ollama local · gratis · sin internet

Corre modelos de IA en tu propio equipo. Ideal para privacidad y para aprender sin pagar por uso. No necesita Node.js.

Descarga oficial de Ollama Instalación Linux Instalación Mac
  1. Windows/Mac: descarga el instalador desde ollama.com/download.
  2. Linux:
    curl -fsSL https://ollama.com/install.sh | sh
  3. Verifica:
    ollama -v
  4. Baja y prueba un modelo pequeño:
    ollama run llama3.2
  5. Para conectar otras herramientas, usa la dirección local:
    http://localhost:11434
Por qué importa: con Ollama, los modelos corren en tu equipo. La primera descarga necesita internet; después puedes usar el modelo descargado sin conexión.
Errores comunes y solución rápida
“command not found” o “no se reconoce el comando”: cierra y vuelve a abrir la terminal; si sigue, revisa que Node, npm o el instalador hayan agregado el programa al PATH.
“permission denied”, EACCES o permisos de npm: evita repetir comandos al azar; usa nvm, corrige la carpeta global de npm o instala con el instalador oficial.
El navegador no abre para iniciar sesión: copia el enlace que muestra la terminal y pégalo manualmente en tu navegador.
La terminal se queda “pensando”: revisa internet, cierra con Ctrl+C y vuelve a ejecutar el comando de verificación.
En Windows algo falla con herramientas de terminal: instala WSL y repite los pasos Linux dentro de Ubuntu/WSL.

Conecta la Suite a tu asistente

Ya con tu asistente instalado, conecta las herramientas. La vía cambia según cuál uses.

Claude Code - plugin o MCP

  1. Pega en Claude:
    Instala el plugin de DevControl de la Suite y verifica que quede conectado como MCP.
  2. Empieza por DevControl. Cuando funcione, repite para devSteps y las demás herramientas.
  3. Verifica pidiéndole: “lista las herramientas MCP disponibles y dime si DevControl aparece”.

Cursor / VS Code - la pestaña

  1. Genera el paquete .vsix de la extensión desde la carpeta de la herramienta. Normalmente se hace con:
    npm install && npx vsce package
  2. Si el proyecto usa otro empaquetador, busca en su package.json un script como package, build o vsix y ejecútalo.
  3. Instala el archivo generado en VS Code/Cursor: abre extensiones, menú de tres puntos, Install from VSIX....
  4. También puedes usar terminal:
    code --install-extension nombre-del-archivo.vsix
  5. Abre la pestaña Suite en la barra lateral y pulsa Conectar.
Verifica: reinicia el editor y confirma que aparece la pestaña Suite. Si no aparece, revisa que instalaste el .vsix correcto.

ChatGPT - empieza por linkAnalizer

  1. Levanta linkAnalizer localmente.
  2. En ChatGPT crea una GPT Action e importa su openapi.yaml.
  3. Verifica ejecutando una acción de prueba con un enlace sencillo.

Codex - por MCP

  1. Agrega DevControl como MCP:
    codex mcp add sp-devcontrol -- sp-devcontrol mcp
  2. Verifica en Codex: pregunta “lista tus servidores MCP y confirma si sp-devcontrol está activo”.

Cómo se trabaja: tú diriges, los agentes ejecutan

Así trabaja un equipo con esta suite: un humano coordina, la IA planifica, los agentes ejecutan tareas acotadas, y se revisa todo. Tú eres el director.

1. Túhumano
Dices qué quieres lograr, en lenguaje normal. Ejemplo: “revisa la seguridad de mi proyecto y arregla lo crítico”.
2. AsistenteClaude/Codex
Planifica y divide el trabajo en tareas. Si son muchas o repetitivas, las puede delegar a agentes locales.
3. Agentesen paralelo
Ejecutan cada tarea: escriben, revisan, prueban o documentan. Pueden correr varios a la vez.
4. DevControlgobernanza
Ningún cambio importante se aplica sin que tú apruebes. La IA no se auto-aprueba.
5. Revisióntú + guía
Revisas resultados, corres pruebas y confirmas. devSteps mantiene el orden del proceso.
Ahorro real: delegar tareas acotadas a agentes locales (opencode/Ollama) puede reducir costos. La suite trae llmgraph, que comprime documentación para que la IA use solo el contexto relevante.
Ejemplo concreto de un encargo

Le dices a tu asistente: “prepara mi proyecto para publicar: revisa seguridad, corre los tests y déjame los pasos para subirlo a GitHub”. El asistente planifica, delega la auditoría a un agente, otro corre los tests, tú apruebas los cambios con DevControl, y te entrega el resultado revisado con comandos listos.

6

Dudas frecuentes

Nunca instalé nada por terminal. ¿Puedo?

Sí. La terminal es una ventana donde pegas comandos. Usa los botones “Copiar”. Si te trabas, copia el error exacto y pégalo en tu chat de IA.

¿Necesito pagar?

No para empezar: Ollama es gratis y local. Claude/Codex pueden requerir plan o API según tu caso. opencode puede funcionar con varios proveedores o con Ollama.

¿Es seguro? ¿La IA hará cosas sin permiso?

La Suite está pensada para control humano. Con DevControl, los cambios importantes pasan por aprobación. Aun así, revisa antes de aceptar.

Me pide una clave, token o contraseña

No la compartas si no sabes qué es. Pregúntale primero a tu asistente “¿qué clave me pide, dónde se crea y cómo se guarda?”. Nunca pegues claves en lugares públicos.

¿Qué hago si algo no coincide con mi pantalla?

Los instaladores cambian. Usa el enlace oficial del producto, busca la opción LTS o estable, y verifica con el comando de versión antes de avanzar.
